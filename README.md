# Минимальная стоимость последовательности 1-покрывающих путей

## Условие задачи

Дано дерево из `n` вершин. Игра состоит из `n - 1` раундов. В каждом раунде
в текущем дереве выбирается путь-маршрут

`v[0], v[1], ..., v[w - 1]`,

в котором вершины могут повторяться. Вершины маршрута считаются белыми, а
остальные вершины — чёрными. Каждая чёрная вершина должна быть смежна хотя бы
с одной белой, то есть маршрут должен 1-покрывать текущее дерево.

Пусть:

- `k = w - 1` — длина маршрута в рёбрах;
- `b` — число чёрных вершин.

Стоимость раунда равна

`3k + 2b + 1`.

После раунда удаляется одна чёрная вершина, смежная с последней вершиной
маршрута. Требуется выбрать маршруты и удаляемые вершины так, чтобы суммарная
стоимость всех `n - 1` раундов была минимальной.

Текущая реализация решает задачу точно для дерева при `n <= 20`. В исходной
общей постановке вместо дерева может быть произвольный связный граф, однако
приведённый код читает ровно `n - 1` ребро, а доказательство ниже использует
единственность путей в дереве.

## Идея алгоритма

Вершины кодируются битовыми масками.

### 1. Кратчайшие пути

Алгоритм Флойда вычисляет расстояние `dist[u][v]`, сами кратчайшие пути и
множества входящих в них вершин. В дереве кратчайший путь между двумя
вершинами единственен.

### 2. Минимальный маршрут одного раунда

`tsp[mask][u]` — минимальная длина маршрута, который:

- начинается в произвольной вершине;
- посещает все вершины из `mask`;
- заканчивается в `u`.

Переход:

`tsp[mask ∪ {v}][v] = min(tsp[mask][u] + dist[u][v])`.

Последовательность кратчайших путей между выбранными вершинами может повторно
проходить через уже посещённые вершины, поэтому получаемый маршрут не обязан
быть простым.

### 3. Минимальная стоимость всех раундов

`answer[tree]` — минимальная стоимость перехода от исходного дерева к
связному множеству оставшихся вершин `tree`.

Для каждого состояния перебираются:

- непустое собственное подмножество белых вершин `white ⊂ tree`;
- конец маршрута `u ∈ white`;
- удаляемая вершина `v`.

Подмножество `white` допустимо, если оно связно и 1-покрывает `tree`:

`tree ⊆ white ∪ N(white)`.

Тогда число чёрных вершин равно `b = |tree \ white|`, а минимальная стоимость
раунда с концом в `u` равна

`roundCost = 3 * tsp[white][u] + 2b + 1`.

Если `v` — чёрный сосед `u`, выполняется переход:

`answer[tree \ {v}] = min(answer[tree \ {v}], answer[tree] + roundCost)`.

Вместе со значениями динамики сохраняются предки, по которым восстанавливаются
все маршруты и удалённые вершины.

## Корректность

**Лемма 1.** Для связного `white` значение `tsp[white][u]` равно минимальной
длине маршрута, посещающего все вершины `white` и заканчивающегося в `u`.

Заменим участки любого допустимого маршрута кратчайшими путями и удалим из
порядка повторные промежуточные посещения, сохранив последнее посещение `u`.
Из неравенства треугольника длина не увеличится, поэтому полученный порядок
рассматривается динамикой. Обратно, каждый переход динамики добавляет
существующий кратчайший путь и строит допустимый маршрут. Так как `white`
связно и граф является деревом, эти пути не выходят за пределы `white`.
Следовательно, найденная длина оптимальна.

**Лемма 2.** Каждый переход `answer` соответствует допустимому раунду.

Связность `white` гарантирует, что маршрут лежит в текущем дереве. Проверка
1-покрытия красит все остальные вершины в чёрный цвет. Удаляемая `v` не входит
в маршрут и смежна с его концом `u`, поэтому правила удаления выполнены.
Стоимость перехода совпадает со стоимостью раунда.

**Теорема.** Минимум среди `answer[{u}]` равен минимальной суммарной стоимости
всех `n - 1` раундов.

Рассмотрим оптимальную последовательность. Её первый раунд присутствует среди
перебираемых переходов, а оставшаяся часть является той же задачей для дерева
без удалённой вершины. По индукции по числу оставшихся вершин динамика
рассматривает всю оптимальную последовательность. По лемме 2 динамика не
создаёт недопустимых последовательностей. Значит, найденный ответ достижим и
не превосходит никакого допустимого решения, то есть глобально оптимален.

В отличие от жадного выбора самого дешёвого текущего раунда, динамика учитывает
влияние удаляемой вершины на все последующие раунды.

## Сложность

- время: `O(2^n n^2 + 3^n(n + m))`;
- память: `O(2^n n)`.

Экспоненциальная сложность ограничивает размер входа значением `n <= 20`.

## Формат ввода и запуск

Ввод:

```text
n
u1 v1
u2 v2
...
u(n-1) v(n-1)
```

Вершины нумеруются от `1` до `n`.

Сборка и запуск:

```bash
g++ -std=c++17 -O2 main.cpp -o tsp-1r
./tsp-1r < input.txt
```

Программа выводит минимальную стоимость, маршрут каждого раунда и удаляемую
вершину.
